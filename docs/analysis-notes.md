# 分析与实施记录

## 2026-07-18 密码库功能合并

实施方向：

1. 新增独立 Pinia 密码库 Store。
2. 密码记录在内存中以对象数组使用，写入 Preferences 前统一使用主密码 AES 加密。
3. 新存储 Key 使用 `my_password_vault_encrypted`，避免与旧明文数据混淆。
4. 首次加载时如发现旧 Key `my_password_manager_data`，解析后加密迁移；成功写入后删除旧明文 Key。
5. 修改主密码后使用内存中的密码记录，以新主密码重新加密写入。
6. 密码备份在设置页使用独立入口，兼容旧项目以主密码加密的数组格式。
7. UI 延续当前 ForMyself 的 Apple 风格和现有全局交互规范。

## 2026-07-18 心情多事件与标签升级

实施方向：

1. 心情记录继续使用扁平数组，但取消日期唯一约束，同一天允许多个独立事件。
2. 每条事件使用 `tags: string[]` 支持多标签，内置工作、学习、家庭、睡眠。
3. 旧记录或导入记录没有 `tags/tag` 时统一补为 `['学习']`。
4. 兼容曾使用单值 `tag` 的记录，导入后规范为 `tags` 并移除旧字段。
5. 自动补记记录增加 `autoFilled` 标记；用户首次补写该日事件时替换占位记录，避免重复统计。
6. 日历按天显示最新事件心情和事件数，列表按事件独立编辑、删除。
7. 月度情绪分布按事件次数计算，连续记录仍按唯一日期计算。

## 2026-07-18 密码库分类、收藏与二次生物识别

实施方向：

1. 密码记录新增 `category` 和 `favorite`，旧数据分别默认补为“未分类”和 `false`。
2. 兼容可能存在的旧别名 `group`、`isFavorite`，统一规范为新字段。
3. 收藏记录在列表中优先展示，并提供分类筛选、只看收藏与分类搜索。
4. 显示明文和复制密码分别执行一次独立生物识别；不可用、取消或失败均拒绝访问。
5. Web 环境不采用插件的模拟成功结果，必须是 Capacitor 原生平台才能发起密码访问授权。
6. 明文显示 30 秒后自动隐藏，页面进入后台或组件卸载时立即隐藏。

验证结果：

- Node 自动化测试 43/43 通过，覆盖旧结构迁移、别名兼容、收藏排序/切换，以及生物识别可用、不可用、取消和插件异常分支。
- Vite 生产构建通过；Android Gradle 构建通过（304 个任务）。
- Android 模拟器成功导入使用测试主密码加密的旧结构备份；两条记录均自动补为“未分类”和未收藏，随后验证了收藏筛选、自定义分类和分类筛选。
- Android 模拟器未录入生物识别时，“显示密码”和“复制密码”均被真实拒绝，密码保持隐藏。
- 模拟器的系统安全录入界面无法完成指纹样本注册，因此成功指纹弹窗分支由自动化桩测试覆盖；仍建议在已录入指纹或面容的真机上做最终交互确认。

## 2026-07-18 体重趋势增强

实施方向：

1. 健康设置增加身高、目标体重、体重变化提醒开关与阈值；兼容原来只保存 `targetWeight` 的设置。
2. BMI 根据最新有效体重和身高计算，仅作为趋势参考并明确不替代专业健康评估。
3. 周平均先合并同一天的多次称重为日均，再按周一至周日计算自然周平均，避免重复称重对周均造成额外权重。
4. 趋势图默认显示周均，同时允许切换每日记录；体重页摘要显示最新记录周均及与前一记录周的变化。
5. 新记录与上一条有效记录的变化达到用户阈值时显示温和页面提示；已有系统通知权限时同步发送本地通知，不使用医疗诊断措辞。
6. 保持旧体重备份数组格式不变；健康设置不混入旧记录，避免破坏既有导入。

验证结果：

- Node 自动化测试 50/50 通过，其中新增 7 项覆盖健康设置迁移、BMI、同日去重周均、无效日期、变化阈值和通知载荷。
- 真实体重备份 14 条在 Android 成功导入并生成 4 个周均分组。
- Android 验证身高/目标保存、BMI/目标差距、周均/每日切换、设置重启持久化均正常。
- Android 真实触发变化提醒，通知中心记录 ID `2201`，应用页面与系统通知同时生效。
- Vite 生产构建和 Capacitor 同步通过；Android `assembleDebug` 构建通过。
